Self-Validating Bayesian Optimization

A framework for learning from prediction errors to improve nanoparticle synthesis optimization

Overview

Traditional Bayesian optimization treats each new data point simply as additional training data, retraining the surrogate model from scratch. While effective, this approach doesn't explicitly learn from its own prediction errors.

Our self-validating approach goes further: it tracks what the model predicted at the time of each recommendation, compares these predictions to actual experimental results, and learns systematic patterns in these errors to improve future predictions.

Standard BO

  • Retrain on all data each iteration
  • No memory of past predictions
  • Errors are implicit in model fit
  • Uncertainty reflects data coverage

Self-Validating BO

  • Explicit prediction history tracking
  • Learn bias correction from errors
  • Calibrate uncertainty estimates
  • Adapt to systematic model limitations

The Cu₃VS₄ Synthesis Problem

Cu₃VS₄ (copper vanadium sulfide) nanoparticles have tunable properties based on synthesis conditions. The goal is to predict and optimize:

  • Particle Size — Average diameter in nanometers
  • GSD — Geometric standard deviation (size distribution width)
  • Squareness — Morphology parameter (1.0 = perfect cube)

Given five controllable synthesis parameters:

  • Temperature — Reaction temperature (220-300°C)
  • Time — Reaction duration (5-120 min)
  • V(acac)₃ — Vanadium precursor amount (0.1-1.0 mmol)
  • DDT — Dodecanethiol volume (0.5-4.0 mL)
  • OAm — Oleylamine volume (1.0-8.0 mL)

Model Architecture

Gaussian Process Regressors

We use separate Gaussian Process (GP) models for each continuous property (Size, GSD, Squareness). GPs provide both predictions and uncertainty estimates, crucial for Bayesian optimization.

f(x) ~ GP(m(x), k(x, x'))

where m(x) is the mean function and k(x, x') is the covariance kernel. We use a Matérn 5/2 kernel with automatic relevance determination (ARD).

Feasibility Classification

Not all synthesis conditions produce viable nanoparticles. We model three binary feasibility criteria using Gaussian Process Classifiers:

  • HasProduct — Did the synthesis produce measurable particles?
  • PhasePure — Is the product single-phase Cu₃VS₄?
  • IsCubic — Is the morphology cubic (vs. multipod)?
P(feasible) = P(HasProduct) × P(PhasePure) × P(IsCubic)

Error Learning System

The key innovation is explicit tracking and learning from prediction errors. When a recommendation is completed, we compute:

error = actual_value - predicted_value

Bias Correction

We fit a secondary GP to the residuals (errors) as a function of the input conditions. This learns systematic biases:

Example

If the model consistently underpredicts size at high temperatures, the error GP learns this pattern and applies a positive correction in that region.

Uncertainty Calibration

We track whether actual results fall within predicted confidence intervals. If the model is overconfident (results often outside intervals), we scale up the uncertainty; if underconfident, we scale down.

calibrated_std = raw_std × calibration_factor

Recommendation Workflow

1

Generate Candidates

Use Latin Hypercube Sampling to create diverse candidate conditions across the parameter space.

2

Predict Properties

For each candidate, predict Size, GSD, Squareness with uncertainties, applying error corrections if available.

3

Estimate Feasibility

Calculate probability that each candidate will produce a viable, phase-pure, cubic product.

4

Compute Acquisition

Combine Expected Improvement (for size targeting) with probability in interval (for GSD/Squareness bounds).

5

Select Diverse Top Candidates

Choose top candidates while ensuring minimum distance between selections to explore different regions.

Acquisition Function

The acquisition function balances exploitation (targeting desired size) with exploration (reducing uncertainty) and feasibility:

acquisition = EI(size) × PI(GSD ≤ 1.3) × PI(Squareness ≥ 0.5) × P(feasible)

Where:

  • EI — Expected Improvement toward target size
  • PI — Probability of being within acceptable bounds

Data Sources

The system tracks three categories of experiments:

  • Imported — Initial dataset from historical experiments (43 experiments)
  • Recommendation — Experiments run based on model suggestions
  • Manual — User-designed experiments added to the dataset

Only recommendation-sourced experiments contribute to error learning, as these are the only ones where we have recorded predictions to compare against.

Minimum Data Requirements

  • 5 successful experiments — Required to build initial GP models
  • 5 completed recommendations — Required to enable error learning and bias correction
